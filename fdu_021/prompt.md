# fdu_021

## Round 1
Create a premium long-scroll single-file `src/index.html` for a **SaaS landing page** called **OrbitDock**.
Concept: a cloud file collaboration suite marketed to young premium consumers and modern creator teams who want fast review loops, elegant sharing, and polished collaboration rituals.
Audience: young premium consumers, creator teams, and collaborative digital workers who want a more polished file-sharing and review experience.
Style direction: **Bento Grid** with references to Aceternity UI / Magic UI-inspired modular storytelling, premium launch pages, and structured product marketing grids.
Component language: bento cards, polished stat tiles, product preview frames, modular CTA stacks, and creator-friendly workflow panels.
Color direction: deep navy, ink black, cool gray foundations with selective bright accents such as cyan, lime, and muted coral.
Typography direction: strong geometric headings with crisp modern sans-serif body copy.
Motion direction: product-like, crisp, modular, and high-signal but still refined.
Build a complete page with these sections: sticky navbar, hero, trust badges, bento feature grid, workflow story, integrations, security and permissions, pricing, testimonials, FAQ, CTA, footer.
Required interactions: button hover states, button active states, counter animations, scroll reveal animations, sticky navbar transition on scroll.
The page should feel like a premium prosumer collaboration product rather than generic cloud storage marketing.

## Round 2
Deepen the layout and content density.
The hero should show a modular product story with elegant preview cards, strong sharing value, and clear emotional plus practical benefit.
Add believable copy around review links, permissions, version history, async feedback, folder rituals, approval flows, and creator-friendly collaboration polish.
Balance conversion structure with product credibility through believable copy, differentiated modules, and workflow storytelling that feels product-real.
Use the bento system to add depth and hierarchy rather than decorative fragmentation.
Keep enough vertical depth for a long full-page screenshot, with multiple strong section transitions and no thin filler sections.

## Round 3
Polish all interaction states and compliance details.
Make hover, active, and focus states clearly visible across cards, buttons, navigation, and CTA modules.
Counters should animate into place when scrolled into view, and the sticky navbar should gain a subtle blur, border, or elevation change after scrolling.
Ensure the layout is responsive for desktop and tablet widths while preserving the modular premium look.
Keep all CSS and JS inline inside `index.html`.
Do not use React, Vue, Svelte, jQuery, or any build step.
Do not reference local images, local fonts, local CSS, or local JS.
Use remote imagery only if needed and inline SVG icons.
Return only the final `src/index.html` content with no explanation.

## Round 4
Final refinement pass.
The result should feel polished, creator-friendly, and product-real while still being highly conversion-oriented.
Avoid commodity storage messaging, bland productivity clichés, and empty bento cards.
Ensure enough depth for a long full-page screenshot and visible interactive states throughout.
Return one complete self-contained `index.html`.
