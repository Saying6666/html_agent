# fdu_016

## Round 1
Create a premium long-scroll single-file `src/index.html` for a **dashboard** called **CourseBoard**.
Concept: a restaurant tasting menu launch translated into a chef-operations dashboard that tracks seatings, pacing, pairings, and guest experience in a bold tactile interface.
Audience: developers and operators who appreciate systems thinking, plus hospitality teams who want a high-personality operational dashboard concept.
Style direction: **Neo-Brutalism** with references to bold editorial tech posters, tactile UI blocks, and playful brutalist layouts with premium discipline.
Component language: thick bordered cards, offset shadows, oversized labels, punchy status chips, bold performance panels, and tactile action modules.
Color direction: high-contrast black and white foundations with curated bursts of acid yellow, cobalt, tomato red, and mint.
Typography direction: assertive display typography with sturdy supporting sans-serif text and strong operational labels.
Motion direction: snappy, tactile, playful, and clearly stateful.
Build a complete page with these sections: sticky navbar, hero summary, service metrics, pacing dashboard, table flow modules, menu chapter tracker, team notes, pricing or demo CTA, FAQ, footer.
Required interactions: hover states, tabs, accordion, scroll reveal, smooth scrolling.
The page should feel like a memorable hospitality operations product rather than a normal POS dashboard.

## Round 2
Deepen the layout and content density.
The hero should feel bold and operational, mixing hospitality energy with clear seatings, pacing, and service-performance signals.
Add believable copy around course timing, guest notes, pairings availability, allergy flags, table turnover, service communication, and menu performance insights.
Use dashboard storytelling that feels operationally real, with comparison cards, activity feeds, status modules, and summary views rather than decorative charts alone.
Make the page feel bold and characterful without losing usability or hierarchy.
Keep enough vertical depth for a long full-page screenshot, with multiple strong section transitions and no thin filler sections.

## Round 3
Polish all interaction states and compliance details.
Tabs should feel crisp and obviously stateful, accordions should expand smoothly with strong contrast, and scroll reveal motion should guide attention without slowing the page down.
Smooth scrolling should make anchor navigation feel premium rather than ornamental.
Ensure the layout is responsive for desktop and tablet widths while preserving the tactile brutalist identity.
Keep all CSS and JS inline inside `index.html`.
Do not use React, Vue, Svelte, jQuery, or any build step.
Do not reference local images, local fonts, local CSS, or local JS.
Use remote imagery only if needed and inline SVG icons.
Return only the final `src/index.html` content with no explanation.

## Round 4
Final refinement pass.
The result should feel bold, useful, and product-real rather than like a novelty visual exercise.
Avoid restaurant clipart energy, fake analytics, and brutalism that kills usability.
Ensure enough depth for a long full-page screenshot and visible interactive states throughout.
Return one complete self-contained `index.html`.
