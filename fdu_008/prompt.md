# fdu_008

## Round 1
Create a premium long-scroll single-file `src/index.html` for an **event or conference page** called **Sprint Assembly**.
Concept: a remote design sprint summit built for creative professionals who want strong facilitation systems, practical collaboration methods, and memorable speaker energy.
Audience: creative professionals, design leads, strategists, and workshop facilitators who want a conference page with personality and real event depth.
Style direction: **Neo-Brutalism** with references to bold editorial tech posters, experimental conference identities, and playful brutalist layouts with premium discipline.
Component language: thick bordered cards, offset shadows, oversized labels, punchy badges, tactile schedule modules, and bold CTA blocks.
Color direction: high-contrast black and white foundations with curated bursts of acid yellow, cobalt, tomato red, and mint.
Typography direction: assertive display typography with sturdy supporting sans-serif text and strong label systems.
Motion direction: snappy, tactile, playful, and clearly stateful.
Build a complete page with these sections: sticky navbar, hero, keynote lineup, agenda, workshop tracks, facilitation playbook highlights, sponsor strip, ticket tiers, FAQ, footer.
Required interactions: hover states, tabs, accordion, scroll reveal, smooth scrolling.
The page should feel like an unforgettable design event identity rather than a generic meetup template.

## Round 2
Deepen the layout and content density.
The hero should use oversized typography, high-contrast badges, a strong event promise, and immediate date or ticket urgency.
Add believable copy around remote whiteboarding sessions, sprint rituals, facilitation frameworks, breakout formats, speaker personalities, and ticket plan differences.
Make the speaker, schedule, workshop, and ticketing sections detailed enough that a real attendee could evaluate the event from this page alone.
Use more section depth, stronger event pacing, and bold but readable brutalist composition throughout.
Keep enough vertical depth for a long full-page screenshot, with multiple strong section transitions and no thin filler sections.

## Round 3
Polish all interaction states and compliance details.
Tabs should feel crisp and obviously stateful, accordions should expand smoothly with strong contrast, and scroll reveal motion should guide attention without slowing the page down.
Smooth scrolling should make anchor navigation feel premium rather than ornamental.
Ensure the layout is responsive for desktop and tablet widths while preserving the bold identity.
Keep all CSS and JS inline inside `index.html`.
Do not use React, Vue, Svelte, jQuery, or any build step.
Do not reference local images, local fonts, local CSS, or local JS.
Use remote imagery only if needed and inline SVG icons.
Return only the final `src/index.html` content with no explanation.

## Round 4
Final refinement pass.
The result should feel bold, memorable, and professionally art-directed while still being usable and conversion-oriented.
Avoid chaotic randomness, meme-heavy copy, and purely decorative brutalist noise.
Ensure enough depth for a long full-page screenshot and visible interactive states throughout.
Return one complete self-contained `index.html`.
