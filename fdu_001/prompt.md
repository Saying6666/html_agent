# fdu_001

## Round 1
Create a premium long-scroll single-file `src/index.html` for a **SaaS landing page** called **CareFlow**.
Concept: a telehealth wellness app for mid-market product teams that want preventive care, mental wellness check-ins, and lightweight team health programs in one platform.
Audience: mid-market product teams, HR leaders, and operations managers evaluating a credible healthcare-adjacent digital product.
Style direction: **Clean Corporate** with references to **Land-book** and modern B2B health-tech websites.
Component language: **Shadcn/ui-like** cards, buttons, tabs, and clean data modules.
Color direction: soft clinical white, cool slate, muted teal, calm blue accents, subtle success green; premium, trustworthy, not playful.
Typography direction: strong modern geometric headings with clean neutral body text; polished enterprise feel.
Motion direction: subtle product-style motion with restrained scroll reveals and elegant state changes.
Build a complete content-rich page with these sections: sticky navbar, hero, trust logos, metrics, core features, care journey workflow, integrations, pricing, testimonials, FAQ, CTA, footer.
Required interactions: button hover states, button active states, counter animations, scroll reveal animations, sticky navbar transition on scroll.
The page must feel like a real 2025–2026 health-tech SaaS product, not a classroom demo.

## Round 2
Deepen the layout and content density.
Make the hero include a strong headline, supporting copy, dual CTAs, product UI mockup framing, and a metrics strip.
Add believable product copy about care plans, asynchronous check-ins, wellness insights, clinician network access, and privacy-first workflows.
Use strong visual hierarchy, professional spacing rhythm, premium cards, polished iconography, and realistic dashboard/app preview compositions.
Expand the workflow section into a step-by-step journey such as onboarding, check-in scheduling, care recommendations, team insights, and reporting.
Pricing should feel enterprise-ready with clear plan differentiation and realistic feature bullets.
Testimonials should sound credible and B2B-specific, not generic startup quotes.
FAQ should address compliance, onboarding, integrations, and reporting.
Keep enough vertical depth for a long full-page screenshot.

## Round 3
Polish all states and compliance details.
Make hover, active, and focus states clearly visible across buttons, nav links, cards, and pricing actions.
Use scroll reveal on major content blocks and animated counters in the metrics area.
The sticky navbar should gain a subtle blur, border, or shadow transition after scrolling.
Ensure the layout is responsive for desktop and tablet widths.
Keep all CSS and JS inline inside `index.html`.
Do not use React, Vue, Svelte, jQuery, or any build step.
Do not reference local images, local fonts, local CSS, or local JS.
Use remote imagery only if needed and inline SVG icons.
Return only the final `src/index.html` content with no explanation.

## Round 4
Final quality pass.
Make sure the page feels polished, trustworthy, premium, and healthcare-adjacent without using problematic medical claims.
Avoid placeholder copy, empty sections, weak cards, and generic gradients.
Ensure the final page is long enough for a full-page screenshot and visibly includes the required interaction patterns.
Return one complete self-contained `index.html` that opens directly in a browser.
