# fdu_005

## Round 1
Create a premium long-scroll single-file `src/index.html` for a **mobile app landing page** called **PulseTrace Mobile**.
Concept: a developer API analytics dashboard delivered as a mobile-first companion experience for enterprise buyers who want real-time visibility into API health, cost, traffic, and incidents.
Audience: enterprise buyers, platform leaders, engineering managers, and technical decision-makers.
Style direction: **Bento Grid** with references to modern product launches, high-end dashboard marketing sites, and modular UI storytelling.
Component language: **Aceternity UI / Magic UI-inspired** bento cards, polished stat tiles, compact product previews, structured CTA modules.
Color direction: deep navy, ink black, cool grays, electric blue, subtle cyan and lime accents for signal highlights.
Typography direction: strong geometric display headings with clean enterprise sans-serif body text.
Motion direction: crisp, product-like, modern, high-signal but not flashy.
Build a complete page with these sections: sticky navbar, hero, app preview showcase, bento feature grid, metrics counters, integrations, security/compliance, testimonials, pricing or demo CTA, FAQ, footer.
Required interactions: hover states, active states, counters, scroll reveal, sticky navbar.
The page should feel like a real premium enterprise mobile product launch page.

## Round 2
Deepen the layout and give the page strong conversion structure.
The hero should combine a clear enterprise value proposition, mobile device mockup framing, a metrics strip, and dual CTA actions.
Use a rich bento grid to highlight API uptime alerts, cost anomalies, request tracing, team notifications, and executive summary views.
Add believable enterprise copy around governance, observability, integrations, and incident response.
Use counters for request volume, monitored endpoints, alert reduction, or time-to-resolution improvements.
Show compatibility with major cloud providers and team workflows.
Make the page visually dense and premium enough for a long screenshot, with multiple strong section transitions.

## Round 3
Polish the interaction system.
Buttons and cards should have strong hover and active feedback.
Counters should animate into place when scrolled into view.
Major sections should use tasteful scroll reveal motion.
The sticky navbar should visibly change background, blur, border, or elevation after scrolling.
Make mobile preview modules feel realistic and polished, not generic rectangles.
Ensure responsiveness for desktop and tablet while preserving the mobile-product emphasis.
Keep all CSS and JS inline in `index.html`.
No frameworks, no local assets, no local CSS or JS, no build step.
Use remote imagery only if needed and inline SVG icons.
Return only the final HTML.

## Round 4
Final quality pass.
The page should feel like a premium enterprise product marketing page with strong modular hierarchy and believable technical value.
Avoid empty bento cards, shallow copy, random gradients, and startup-template clichés.
Ensure enough content depth and visible interaction variety for a long full-page screenshot.
Return one complete self-contained `index.html` only.
