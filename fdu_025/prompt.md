# fdu_025

## Round 1
Create a premium long-scroll single-file `src/index.html` for a **mobile app landing page** called **CipherPocket**.
Concept: a cybersecurity consulting studio translated into a mobile-first advisory companion app for mid-market product teams who want alerts, audits, and security decisions in motion.
Audience: mid-market product teams, engineering managers, and operators who want mobile-accessible security visibility with premium trust cues.
Style direction: **Clean Corporate** with references to Land-book, premium B2B mobile-product launches, and polished enterprise software marketing.
Component language: clean phone mockup frames, Shadcn/ui-like content cards, quiet security panels, premium CTA sections, and trust-focused data modules.
Color direction: soft white, cool slate, muted blue, restrained teal, and graphite accents with strong enterprise contrast.
Typography direction: strong geometric display headings paired with clean neutral sans-serif body text.
Motion direction: subtle product-style motion with restrained scroll reveals and elegant state changes.
Build a complete page with these sections: sticky navbar, hero, mobile app preview showcase, advisory modules, alert and audit cards, workflow walkthrough, testimonials, pricing or demo CTA, FAQ, footer.
Required interactions: button hover states, button active states, counter animations, scroll reveal animations, sticky navbar transition on scroll.
The page should feel like a believable enterprise companion product rather than a generic app mockup on a white background.

## Round 2
Deepen the layout and content density.
The hero should show a credible security companion app with strong mobile UI framing, trust messaging, and clear value for busy product teams.
Add believable copy around incident alerts, audit summaries, policy checklists, advisory sessions, risk snapshots, and mobile decision support for leadership teams.
Make the phone UI modules feel believable, product-specific, and polished rather than like generic placeholder screens.
Use strong visual hierarchy, richer content modules, and more structure around trust, workflow, and conversion.
Keep enough vertical depth for a long full-page screenshot, with multiple strong section transitions and no thin filler sections.

## Round 3
Polish all interaction states and compliance details.
Make hover, active, and focus states clearly visible across buttons, cards, nav links, and primary CTAs.
Counters should animate into place when scrolled into view, and the sticky navbar should gain a subtle blur, border, or elevation change after scrolling.
Ensure the layout is responsive for desktop and tablet widths while preserving the enterprise-ready mobile feel.
Keep all CSS and JS inline inside `index.html`.
Do not use React, Vue, Svelte, jQuery, or any build step.
Do not reference local images, local fonts, local CSS, or local JS.
Use remote imagery only if needed and inline SVG icons.
Return only the final `src/index.html` content with no explanation.

## Round 4
Final refinement pass.
The result should feel polished, trustworthy, and product-real while still maintaining strong mobile-first conversion clarity.
Avoid red-alert fear marketing, fake hacker visuals, and shallow mobile screens.
Ensure enough depth for a long full-page screenshot and visible interactive states throughout.
Return one complete self-contained `index.html`.
