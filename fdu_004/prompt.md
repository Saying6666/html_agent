# fdu_004

## Round 1
Create a premium long-scroll single-file `src/index.html` for a **portfolio site** called **SyncVault**.
Concept: a dark-mode portfolio-style showcase for a cloud file collaboration suite aimed at developers and operators.
Audience: developers, DevOps teams, platform engineers, and technical buyers who care about reliability, speed, auditability, and workflow clarity.
Style direction: **Dark Mode** with references to modern developer tooling sites, Awwwards-inspired product storytelling, and premium SaaS portfolios.
Component language: layered dark cards, code-adjacent UI panels, tabbed showcases, structured FAQ accordions, polished product modules.
Color direction: near-black, graphite, deep indigo, cool steel gray, electric cyan accents used sparingly.
Typography direction: sharp modern headings with highly readable technical body text.
Motion direction: subtle but precise, product-like, technical, controlled.
Build a complete page with these sections: navbar, hero intro, selected work grid or workflow grid, project spotlight, process section, integrations or stack compatibility, testimonials, FAQ accordion, contact/footer CTA.
Required interactions: hover states, tabs, accordion, scroll reveal, smooth scrolling.
The page should feel like a real premium collaboration platform presented through a portfolio-style narrative.

## Round 2
Expand the story and make the page feel sophisticated.
In the hero, communicate version control clarity, secure collaboration, fast review cycles, and operational visibility.
Use a selected work or workflow grid to showcase use cases such as design review, incident collaboration, approvals, asset sync, and audit trails.
Create a project spotlight section that feels like a case study with layered UI previews and technical outcome highlights.
The process section should explain how teams move from upload to review to approval to deployment-ready handoff.
Tabs should reveal different product perspectives such as engineering, design, operations, and security.
Testimonials should sound technical and believable.
Add enough depth and layered layout variety for a long screenshot.

## Round 3
Polish micro-interactions and behavior.
Make tabs feel crisp and clearly stateful.
Use accordion behavior for the FAQ with smooth expansion and strong contrast.
Apply scroll reveal to spotlight blocks, workflow modules, and metrics-style highlights.
Use smooth scrolling for nav anchor movement.
Add tasteful hover behavior on cards, links, and CTA elements.
Ensure dark surfaces remain readable and premium rather than muddy.
Keep all CSS and JS inline in `index.html`.
No build step, no frameworks, no local assets, no extra files.
Use inline SVG icons and remote imagery only if truly needed.
Return only the final HTML.

## Round 4
Final refinement pass.
The result should feel like a premium developer-tool brand site with portfolio storytelling, not a generic dark SaaS template.
Avoid filler code blocks, fake terminal gimmicks, and shallow one-line sections.
Ensure enough content depth, interaction richness, and polished contrast for a full-page screenshot.
Return one complete self-contained `index.html`.
